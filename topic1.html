Variables

Variables are used to store information to be referenced and manipulated in a computer program. They also provide a way of labeling data with a descriptive name, so our programs can be understood more clearly by the reader and ourselves. It is helpful to think of variables as containers that hold information.

It is important to declare the variables used in the program (if global) or function (if local) on top and use them later. It makes it easier for maintaining, and considered a best practice.

        var car, bus, pedestrian;
        function myFunction(){
        var first, second, i, y, x;
        // ... do something
        }

With the EcmaScript 6 version it is also better to pick let or const.
Let has block scope whereas var has function scope. Using let instead of var means using the variables in the place where they were declared which produces the less prone to error code.

Const.The const keyword was introduced in es6 to allow immutable variables. That is, variables whose value cannot be modified once assigned.

Hoisting.Hoisting is a JavaScript mechanism where variables and function declarations are moved to the top of their scope before code execution.

Assigning value:

        let x, y z;
        x = 10
        for (y = 0; x < something; y++){
        // ... do something
        }
